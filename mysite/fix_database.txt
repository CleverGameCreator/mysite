Инструкция по восстановлению базы данных:

1. Сначала остановите сервер разработки

2. Сделайте резервную копию вашей текущей базы данных:
   - Скопируйте файл db.sqlite3 в db.sqlite3.backup

3. Удалите существующую базу данных:
   - Удалите файл db.sqlite3

4. Отредактируйте файл mysite/settings.py:
   - Удалите или закомментируйте строки с MIGRATION_MODULES

5. Удалите миграции для модели Comment:
   - Удалите файл blog/migrations/0003_create_comment_model.py

6. Создайте новую миграцию:
   - python manage.py makemigrations blog

7. Примените миграции:
   - python manage.py migrate

8. Создайте суперпользователя:
   - python manage.py createsuperuser

9. Запустите сервер разработки:
   - python manage.py runserver

Эти шаги полностью сбросят вашу базу данных, создадут новые таблицы и применят все миграции корректно. Имейте в виду, что все существующие данные будут потеряны.
